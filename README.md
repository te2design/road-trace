# Road Trace

地図で範囲を選ぶと、その範囲の**道路を白線・横断歩道つきのSVGマップ**にし、さらに
**デジタルツイン風の3Dマップ（道路・建物・地形）として表示・glTF書き出し**までできるWebアプリです。

## 🌐 使ってみる → **https://road-trace.pages.dev**

ブラウザで上のURLを開くだけで使えます（インストール不要）。すべての処理はブラウザの中だけで動き、作業内容が外部に送信されることはありません。

- 道路・建物データ: **OpenStreetMap**（Overpass API経由）
- 航空写真・標高・背景地図: **国土地理院 地理院タイル**
- 場所検索: **Nominatim / OpenStreetMap**

> ※ Googleマップの地図・航空写真を「なぞって別の地図を作る」ことは規約で禁止されているため、本アプリはトレースが許可されているデータ（OSM＋地理院）だけを使っています。

## 使い方（基本の流れ）

1. **場所を探す**（任意）… 地図左上の検索ボックスに「渋谷駅」「東京タワー」などを入れて検索 → 候補をクリックすると地図がそこへ移動します
2. **範囲を決める** … 「地図上でドラッグして範囲指定」で枠を描く（緯度経度の数値入力・お試し範囲も可）
3. **データ取得 → SVG生成** … OpenStreetMapから道路・建物を取得し、白線・横断歩道を自動生成
4. **② SVGマップ**タブで確認 → 「SVGをダウンロード」（**1ユニット=1mの実寸データ**、レイヤー分け済み）
5. **手描きトレース**（任意）… 背景を航空写真にして、写真に写っている白線・矢印などをなぞって追加（写真のズレ補正つき）
6. **AI検出**（実験・任意）… 表示中の範囲から横断歩道・停止線らしい場所を自動で探して候補提案
7. **③ 3Dマップ** … 「3Dを生成」で道路＋建物＋地形を3D表示 → 「glTF(.glb)書き出し」でBlender等へ

作業内容はブラウザに自動保存され、「プロジェクト保存(JSON)」で書き出し・共有もできます。

## 自動生成される路面標示（日本の標準寸法）

| 標示 | 寸法 |
|---|---|
| 車道外側線 | 白実線 幅15cm |
| 中央線 | 白破線15cm（5m/5m）、4車線以上は実線 |
| 車線境界線 | 白破線15cm（6m/9m） |
| 停止線 | 白実線 幅45cm（横断歩道の手前2m） |
| 横断歩道 | 縞幅45cm・間隔45cm・奥行き4m（縞は車の進行方向に平行） |
| 進行方向矢印 | turn:lanesタグから自動配置（約5m、近似形状） |
| ダイヤマーク | 横断歩道の約30m手前（信号なし横断歩道） |

実際の白線の本数・種類は場所により異なります。地図データに無い部分は手描きトレースで補ってください。

## 3D・glTFについて

- **表示スタイル**を2種類から選べます（3Dマップ画面のツールバー）:
  - **デジタルツイン（透過・既定）**: 暗い背景に、半透明の建物＋発光する青い輪郭線、光る道路網、足元のグリッド。ホログラム調
  - **リアル**: 不透明で実物に近い見た目（建物は用途別の色分け）
- 座標はメートル実寸（Y-up）。BlenderにインポートするとZ-upに自動変換されます
- ノード階層: Terrain / Roads / Grid / Markings / Crosswalks / Buildings（＋輪郭線）/ Traces
- 「航空写真を地面に貼る」をONにして書き出すと、写真テクスチャも.glbに埋め込まれます

### 書き出した.glbの見え方について（デジタルツイン）

「街そのもの」（建物・透過・青い輪郭線・道路・標示・グリッド・色）は.glbに保存されます。ただし次の3つは**このアプリの画面だけ**の演出で、ファイルには入りません（他のソフトで開くと少し違って見えます）:

1. **暗い背景・遠景のかすみ（フォグ）** … 他のビューアは各自の背景（白／灰など）で表示します
2. **発光のにじみ** … 線は「単色の半透明」として保存されます
3. **色味の調整（トーンカーブ）** … 他ソフトではやや明るめ・あっさりめに見えることがあります

形・透明感・輪郭線・青系の色は忠実に保存されます。完全に同じ見た目を共有したいときは、アプリのスクリーンショットや短い画面録画を.glbに添えてください。

## データ出典（公開・配布時は必ず添えてください）

```
地図データ © OpenStreetMap contributors（ODbL）
航空写真・標高: 地理院タイル（国土地理院）
場所検索: Nominatim / OpenStreetMap
```

生成したSVGには上記の出典が自動で埋め込まれます。ゲーム等への組み込みも上記クレジット表記で可能です（OSMデータの大規模再配布はODbLの継承条件に注意）。

## 既知の制限

- 航空写真・標高（地理院）は**日本国内のみ**。海外は道路線（OSM）は出ますが写真・3D地形は出ません
- 中庭のある建物（ドーナツ型）は外周のみで表現されます
- 高架・トンネルの立体交差は簡易表現です（橋は両端の高さを直線でつなぐ／トンネルは3D非表示）
- AI検出は写真の解像度（約0.3〜0.6m/画素）の制約上、幅15cmの白線は検出できません。横断歩道・停止線・大きな標示が対象です
- 道路取得（Overpass）・場所検索（Nominatim）は無料の共有サービスです。各ユーザーのブラウザから直接呼ぶため負荷は分散されますが、混雑時は遅くなる/一時的に制限されることがあります
- 等距円筒近似のため、2km四方で最大約0.5m程度の位置誤差があります

## 開発・ローカルで動かす（開発者向け）

ビルド工程はありません（HTML + ES modules、ライブラリはCDN読み込み）。`index.html` を直接ダブルクリックしても動かないため、簡易サーバー越しに開きます。

```bash
git clone https://github.com/te2design/road-trace.git
cd road-trace
python3 -m http.server 8765      # → http://localhost:8765 を開く
```

- macOSなら `start.command` をダブルクリックでも起動できます（初回は右クリック→開く）
- 開発中にJSの変更が反映されないときは、キャッシュ無効サーバー `python3 serve.py 8765` を使ってください

### デプロイ

Cloudflare Pages（直接アップロード）でホスティングしています。

```bash
wrangler pages deploy . --project-name road-trace --branch main
```
