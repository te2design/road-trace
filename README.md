# machi-trace（街トレース）

地図上で範囲を指定すると、その範囲の**道路を白線・横断歩道つきのSVGマップ**にし、
さらに**3Dマップ（道路・建物・地形）として表示・glTF書き出し**までできるWebアプリです。

- 道路・建物データ: **OpenStreetMap**（Overpass API経由）
- 航空写真・標高: **国土地理院 地理院タイル**
- すべてブラウザの中だけで動きます（外部に作業内容を送信しません）

> Googleマップの地図・航空写真を「なぞって別の地図を作る」ことは規約で禁止されているため、
> 本アプリはトレースが許可されているデータ（OSM＋地理院）だけを使います。

## 起動方法

1. `start.command` を**ダブルクリック**
   - 初回は「開発元が未確認」と出ることがあります → ファイルを**右クリック → 開く**で起動できます
   - ターミナルの黒いウィンドウが開き、自動でブラウザにアプリが表示されます
2. 終了するときは黒いウィンドウを閉じます

※ ダブルクリックで開けない場合: ターミナルで
`cd このフォルダ && python3 -m http.server 8765` を実行し、ブラウザで http://localhost:8765 を開いてください。
（`index.html` を直接ダブルクリックしても動きません。ローカルサーバーが必要です）

## 使い方（基本の流れ）

1. **場所を探す**（任意）… 地図左上の検索ボックスに「渋谷駅」「東京タワー」などを入れて検索 → 候補をクリックすると地図がそこへ移動します
2. **範囲を決める** … 「地図上でドラッグして範囲指定」を押して枠を描く（数値入力・お試し範囲も可）
2. **データ取得 → SVG生成** … OpenStreetMapから道路・建物を取得し、白線・横断歩道を自動生成
3. **② SVGマップ**タブで確認 → 「SVGをダウンロード」（**1ユニット=1mの実寸データ**、レイヤー分け済み）
4. **手描きトレース** … 背景を航空写真にして、写真に写っている白線・矢印などをなぞって追加
   - 写真と道路線がずれている場合は「写真ズレ補正」で合わせられます
5. **AI検出（実験）** … 表示中の範囲から横断歩道・停止線らしき場所を自動で探して候補を提案
6. **③ 3Dマップ** … 「3Dを生成」で道路＋建物＋地形の3D表示 → 「glTF(.glb)書き出し」でBlender等へ

## 自動生成される路面標示（日本の標準寸法）

| 標示 | 寸法 |
|---|---|
| 車道外側線 | 白実線 幅15cm |
| 中央線 | 白破線15cm（5m/5m）、4車線以上は実線 |
| 車線境界線 | 白破線15cm（6m/9m） |
| 停止線 | 白実線 幅45cm（横断歩道の手前2m） |
| 横断歩道 | 縞幅45cm・間隔45cm・奥行き4m（縞は車の進行方向に平行） |
| 進行方向矢印 | turn:lanesタグから自動配置（約5m、近似形状） |
| ダイヤマーク | 横断歩道の約30m手前（信号なし横断歩道） |

実際の白線の本数・種類は場所により異なります。地図データに無い部分は手描きトレースで補ってください。

## 3D・glTFについて

- **表示スタイル**を2種類から選べます（3Dマップ画面のツールバー）:
  - **デジタルツイン（透過・既定）**: 暗い背景に、半透明の建物＋発光する青い輪郭線、光る道路網、足元のグリッド。デジタルツイン／ホログラム調
  - **リアル**: 不透明で実物に近い見た目（建物は用途別の色分け）
- 座標はメートル実寸（Y-up）。BlenderにインポートするとZ-upに自動変換されます
- ノード階層: Terrain / Roads / Grid / Markings / Crosswalks / Buildings（＋輪郭線）/ Traces
- 「航空写真を地面に貼る」をONにして書き出すと、写真テクスチャも.glbに埋め込まれます

### 書き出した.glbの見え方について（デジタルツイン）

「街そのもの」（建物・透過・青い輪郭線・道路・標示・グリッド・色）は.glbに保存されます。ただし次の3つは**このアプリの画面だけ**の演出で、ファイルには入りません（他のソフトで開くと少し違って見えます）:

1. **暗い背景・遠景のかすみ（フォグ）** … 他のビューアは各自の背景（白／灰など）で表示します
2. **発光のにじみ** … 線は「単色の半透明」として保存されます
3. **色味の調整（トーンカーブ）** … 他ソフトではやや明るめ・あっさりめに見えることがあります

形・透明感・輪郭線・青系の色は忠実に保存されます。完全に同じ見た目を共有したいときは、アプリのスクリーンショットや短い画面録画を.glbに添えてください。

## 出典表記（必ず添えてください）

生成したSVG・glTF・画像を公開・配布する場合:

```
地図データ © OpenStreetMap contributors（ODbL）
航空写真・標高: 地理院タイル（国土地理院）
場所検索: Nominatim / OpenStreetMap
```

ゲーム等への組み込みも上記クレジット表記で可能です（OSMデータの大規模再配布はODbLの継承条件に注意）。

## 既知の制限

- 中庭のある建物（ドーナツ型）は外周のみで表現されます
- 高架・トンネルの立体交差は簡易表現です（橋は両端の高さを直線でつなぐ／トンネルは3D非表示）
- 標識ベースの細かい規制（はみ出し禁止の黄線区間など）はデータに無いため既定値で描画します
- AI検出は写真の解像度（約0.3〜0.6m/画素）の制約上、幅15cmの白線は検出できません。横断歩道・停止線・大きな標示が対象です
- Overpass API（無料の共有サーバー）が混雑していると取得に時間がかかることがあります
- 等距円筒近似のため、2km四方で最大約0.5m程度の位置誤差があります

## フォルダ構成

```
machi-trace/
├── index.html       … 画面
├── css/style.css    … 見た目
├── js/
│   ├── main.js      … 全体の結線
│   ├── config.js    … 設定値（道路幅・標示寸法・色）
│   ├── geo.js       … 座標変換・幾何計算
│   ├── overpass.js  … OSMデータ取得
│   ├── roadmodel.js … 道路ネットワーク構築
│   ├── markings.js  … 路面標示の自動生成（SVG/3D共通）
│   ├── svgrender.js … SVG出力
│   ├── mapui.js     … 地図・範囲選択・写真ズレ補正・検索ピン
│   ├── trace.js     … 手描きトレース
│   ├── dem.js       … 標高・航空写真タイル
│   ├── three3d.js   … 3D表示（デジタルツイン／リアル）・glTF書き出し
│   ├── detect.js    … AI検出（実験）
│   └── storage.js   … 保存・読み込み
├── start.command    … 起動用（ダブルクリック）
└── serve.py         … 開発・検証用サーバー（キャッシュ無効化。通常利用は不要）
```
